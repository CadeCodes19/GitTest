What?

What is GIT?

Git is a powerful version control system that helps you track changes in your files over time. It's like a time machine for your projects, allowing you to revert to previous versions, experiment with different ideas, and collaborate with others seamlessly.

Why?

Why use GIT?

As defined above GIT is a powerful version control system, this allows:

  • Tracking of Changes Overtime
  • Collaborative Development
  • Experimentation and Risk Mitigation
  • Open Source Contribution
  • Back-up and Recovery
  • Efficient Workflows

How?

How to use GIT?

  1. Initialize a repository (git init)
  2. Add your files (git add .)
  3. Commit your changes (git commit -m 'here goes your message')
  4. This step is optional but creating a new branch (git branch -b 'name of your branch')
  5. Switching from branches (git checkout 'name of your branch')
  6. Merging the added branches (git merge main)
  7. Push origin to main (git push origin main)(git remote add origin https://github.com/CadeCodes19/GitTest.git)
